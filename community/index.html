Requesting New Icons

New icons start as requests by the Roasted Husband community on GitHub. Want to request a new icon? Here are some things to keep in mind:

  1. Please be nice. Roasted Husband is a happy place.
  2. Please search to see if your icon request already exists. If a request is found, please +1 that one.
  3. Please make requests for single icons, unless you are requesting a couple of strictly related icons (e.g., thumbs-up/thumbs-down).
  4. Please and thank you if you include the following:
    • Title your new issue Icon request: rh-name (e.g., Icon request: rh-lotus-root).

询问新的图标

在Roasted Husband community on GitHub中有些图标我们没有创造，但是您需要？那么请做以下的事情

  1. 请您不要急躁，Roasted Husband 是一个快乐的网站.
  2. 请您先在我们的网站上搜寻一下您需要的图标是否已经存在，如果已经存在，请在图标上＋1

  4. 如果您做能做以下的事情，我们将会很感激
    • 在新的问题中新命名您想要的图标: Icon request: rh-name (例如, Icon request: rh-lotus-root)

Reporting Bugs

Found a problem with Roasted Husband? Feel free to submit an issue on the GitHub project. But please keep the following in mind:

  1. Please be nice. Roasted Husband is a happy place.
  2. Please search to see if your bug was already reported.
  3. Before opening any issue, please read the generic issue guidelines, by Nicolas Gallagher.
  4. After doing everything above, feel free to submit an issue.

汇报问题

有问题？请您不要犹豫，我们将会很高兴提高的我们的GitHub产品

  1. 请您不要急躁，Roasted Husband 是一个快乐的网站.
  2. 请您先在我们的网站上搜寻一下您发现的问题是否已经被提交了
  3. 在提交问题汇报之前，请先阅读一下Nicolas Gallagher的问题指引.
  4. 如果您已经做了以上所有的事情，请您快乐的提交问题汇报.
